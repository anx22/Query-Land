# UX Flows

> UI schlank; MCP gleichrangig (Master §3.3). UI = Übersicht, Umsetzung läuft agentisch (§2.4, §4.4).
> Marke: **Query-Land**. Begriffe laut UX-Sprint Teil 1 §4 (Chance, Quell-Verknüpfung, Indexierbarkeit).

## Hauptnavigation
Übersicht · Projekte · Technical Audit · URL-Dossier · Keywords & Rankings · Content & Chancen · Backlinks · Reports · KI-Sichtbarkeit · Einstellungen

## Schlüssel-Screens
- **Übersicht** — Visibility-Verlauf, Health Score, Positions-Verteilung, Top-Chancen, offene Risiken, letzte Crawls/Reports. (UX-1 ✅ — echte Daten, keine Demo-Fixtures.)
- **URL-Dossier** — eine URL als vollständiges SEO-Objekt: Crawlstatus, Indexierbarkeit, GSC-Leistung, Rankings, Queries, interne/externe Links, Content-Fit, Performance, Issues, Chancen, **Quell-Verknüpfung**, Historie.
- **Content & Chancen (Opportunity Board)** — priorisierte Maßnahmen, Filter (Typ/Projekt/URL-Gruppe/Impact/Aufwand/Status), Evidenz-Drawer, Validierungs-Drawer.
- **Technical Audit** — Crawl-Runs, Indexierbarkeits-Funnel, Issue-Gruppen, URL-Explorer, Crawl-Diff.
- **Content Workspace** — _ausstehend (Scope-Entscheidung UX-7):_ Chancen, Briefings, Refresh-Kandidaten, interne Linkvorschläge, Snippet-Vorschläge. Noch kein eigener Screen.

## Erklär-Infrastruktur (UX-9 ✅)
- **`/glossar`** — Begriffsliste + Konfidenz-Legende; einzige Quelle für alle Tooltips.
- **`/kit`** — Showcase aller geteilten Komponenten in allen Zuständen (Abnahme + lebende Doku).
